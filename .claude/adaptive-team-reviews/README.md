# Reviews

Reviewer findings land here, one file per reviewer per review cycle:

`<story-id>/<task-id>/<reviewer>-<cycle>.md`

Where `<reviewer>` is one of `architect`, `sdet`, `llm-expert`, `database`.

Consult-mode analyses land under `consult/<short-slug>/<reviewer>.md`.

The PM does **not** read these files — it sees only the strict verdict from each reviewer. The dev reads them when incorporating feedback. Retained for later `/adaptive-team-learning-moment` diagnosis; purge when the session is complete or after lessons are captured.
